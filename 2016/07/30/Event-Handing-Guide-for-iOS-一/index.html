About Events in iOS

Users manipulate(操控) their iOS devices in a number of ways,such as touching the screen or shaking the device. iOS interprets when and how a user is manipulating the hardware and passes this information to your app. The more your app responds to actions in natural and intuitive ways, the more compelling the experience is for the user.
用户有很多方式操控iOS设备,如触摸屏幕、晃动设备。当用户操控设备时，iOS解释并传递信息给程序。程序以更自然和直观的方式响应用户的操控，就会给用户更好的体验。

At a Glance

Events are objects sent to an app to inform it of user actions. In iOS,events can take many forms:Multi-Touch events,motion events,and events for controlling multimedia. This last type of event is known as a remote control event because it can originate(来自，创造) from an external accessory.
事件是被发送给程序用来通知程序用户动作的对象。在iOS中，事件可以有很多形式：多点触摸事件、运动事件、用来控制多媒体的事件。用来控制多媒体的事件被称为远程控制事件，因为它来自于外部的附件。

UIKit Makes It Easy for Your App to Detect(发现，查明) Gestures

iOS apps recognize combinations(组合) of touches and repond to them in ways that are intuitive to users,such as zooming in on content in response to a pinching gestue and scolling through content in response to a flicking(轻打，轻弹) gesture. In fact,some gestures are so common that they are built in to UIKit. For example,UIControl subclasses,such as UIbutton and UISlider,respond to specific gestures-a tap for a button and a drag for a slider. When you configure(配置，设定) these controls,they send an action message to a target object when that touch occurs. You can also employ（使用、雇用） the target-action mechanism on views by using gesture recognizers. When you attach a gesture recognizer to a view,the entire view acts like a control – reponding to whatever gestue you specify.
iOS程序识别一组触摸，并且以直观的方式响应它们，如用内容的缩放来响应捏合手势，滚动内容来响应flick手势。事实上，一些手势很常见以至于它们被内置于UIKit中。例如，UIControl的子类，像UIButton、UISlider,它们可以响应特定的手势 – 按钮的点击和滑块的拖动。当你配置这些控件后，触摸时它们会给target对象发送一个action消息。在view上可以通过使用gesture recognizer来使用target-action机制。当你将手势识别器添加到view上时，整个view会像控件一样 – 响应你指定的任何手势。

Gesture recognizers provide a higher-level abstraction for complex(复杂的，合成的) event handing logic. Gesture recognizers are the preferred way to implement touch-event handing in your app because gesture recoginzers are powerful,reusable,and adaptable. You can use one of the built-in gesture recogizers and customize its behavior. Or you can create your own gesture recognizer to recognize a new gesture.
Gesture recognizers为复杂的事件处理逻辑提供了高度的抽象。Gesture recognizers是实现程序中touch-event处理的首选方式，因为gesture recognizers是强大的，可从用的，适应性强的。你可以用内置的手势识别并且自定义它的行为。或者你可以创建自己的手势识别器来识别一个新的手势。

Relevant Chapter:Gesture Recognizers

An Event Travels(旅行，行进) Along a Specific Path Looking for an Object to Handle It(事件沿着指定的路径寻找处理它的对象)

When iOS recognizes an event,it passes the event to the initial object that seems most relevant for handling that event,such as the view where a touch occurred. If the initial object cannot handle the event,iOS continues to pass the event to object with greater scope until it finds an object with enough context to handle the event. This sequence of objects is known as a responder chain,and as iOS passes events along the chain,it also transfers（转移，传递）the reponsibility of responding to the event. This design pattern makes event handing cooperative(合作的) and dynamic.
iOS识别事件时后，会将事件传递给最初的似乎最相关的对象来处理事件，像发生触摸的View。如果最初的对象不能处理事件，iOS继续传递事件给更大范围内的对象，知道找到了可以处理事件的对象。这个顺序被称为响应者链，iOS沿着这条链传递事件，并且传递响应事件的责任。这种设计模式协作的动态的处理事件。

Relevant Chapter:Event Delivery(投递):The Responder Chain

A UIEvent Encapsulates(封装) a Touch,Shake-Motion,or Remote-Control Event

Many events are instances of the UIKit UIEvent class. A UIEvent object contains information about the event that your app uses to decide how to respond to the event. As a user action occurs – for example,as fingers touch the screeen and move across its surface – iOS continually(持续地) sends event objects to an app for handling. Each event object has a type – touch,”shaking” motion,or remote control – and a subtype.
许多事件是UIEvent类的实例。一个UIEvent对象包含了你的程序使用的事件的信息来决定如何响应事件。当用户动作发生时 – 例如，手指触摸到屏幕，并在在其表面移动 – iOS持续地给app发送event objects。事件有自己的类型 – 触摸，摇晃，远程控制 – 以及一个子类型。

Relevant Chapters:Multitouch Event, Motion Events,and Remote Controle Events

An App Receives Multitouch Events When Users Touch Its Views

Depending(依赖，取决于) on your app, UIKit controls and gesture recognizers might be sufficient(足够的) for all of your app’s touch event handing. Even if your app has custom views,you can use gesture recognizers. As a rule of thumb(拇指，以拇指拨弄), you write your own custom touch-event handling when your app’s reponse to touch is tightly(紧紧的) coupled with the view itself,such as drawing under a touch. In these cases,you are responsible for the low-level event hanfding. You implement the touch methods,and within these methods,you analyze raw touch events and respond appropriately(适当地).
取决于你的程序，UIKit控件和手势识别器或许对你的APP触摸事件的处理来说是足够的。即使你的程序有自定义的视图，你可以用手势识别器。作为一个经验法则，当你的程序对触摸的响应与视图本身紧密结合，如触摸下绘制，你写你自己的自定义触摸事件处理。在这些情况下，你对底层事件的处理负有责任。你实现触摸方法，在这些方法的内部，你分析未加工的触摸事件并且恰当的响应。

Relevant Chapter:Multitouch Events(多点触摸事件)

An App Receives Motion Events When Users Move Their Devices

Motion Events provide information about the device’s location,orientation,and movement. By reacting(反应) to motion events,you can add subtle敏锐的,yet powerful features to your app. Accelerometer(加速计) and gyroscope(陀螺仪) data allow you to detect tilting(倾斜),rotating,and shaking.
运动事件提供有关设备的位置，方向和运动信息。通过对运动事件反应，你可以为你的程序添加敏锐的强大的特征。加速计和陀螺仪可以让你发觉倾斜，旋转和摇晃。

Motion events come in different forms,and you can handle them using diffrent frameworks. When users shake the device,UIkit delivers a UIEvent object to an app. If you want your app to receive high-rate,continuous accelerometer and gyroscope data,use the Core Motion framework.
运动事件有不同的形式，你可以使用不同的框架来处理它们.当用户晃动设备时，UIKit向APP发送一个事件。如果你希望你的应用接受高速率的持续的加速计和陀螺仪数据，使用Core Motion框架。

Relevant Chapter:Motion Events

An App Receives Remote Control Events When User Mainpulate Mutimedia Controls

iOS controls and external accessories send remote control events to an app. iOS controls and external accessories send remote control events to an app. Handle multimedia remote control events to make your app responsive to these types of commands.
ios控件和外部的配件向APP发送远程控制事件。这些事件允许用户控制音频和视频，如通过耳机调整音量。处理多媒体远程事件可以让你的程序积极响应这些类型的命令。

Relevant Chapter:Remote Control Events

See Also

In the same way that iOS devices provide touch and device motion data,most iOS devices have GPS and compass hardware that generates low-level data that your app might be interested in.
与iOS设备提供触摸和运动数据一样，大多数iOS设备都有GPS和指南针设备来生成应用可能感兴趣的底层数据。
Location and Maps Programming Guide discusses how to receive and handle location data.
Location and Maps Programming Guide讨论了如何接受和处理位置数据。

For advanced gesture recognizer techniques such as curve smoothing and applying a low-pass filter,see WWDC 2012:Building Advanced Gesture Recogizers.
对于高级的手势识别技术，如曲线平滑，和施加一个低通滤波器，看WWDC2012:Building Advanced Gesture Recogizers.

Many sample code projects in the iOS Reference Library have code that uses gesture recognizers and handles events.Among these are the following projects:
在iOS Reference Library中的许多例子项目都有使用手势识别器以及处理事件的代码。这其中有以下项目：

  • Simple Gesture Recognizers is a perfect starting point for understantding gesture recognition.This app demonstrates how to recognize tap,swipe,and rotate gestures.The app responds to each gesture by displaying and animating an image at touch location.
    Simple Gesture Recognizers是一个理解手势识别的完美的着手点。程序演示说明了如何识别tap、swipe、rotate gestures.应用通过在触摸的位置显示和动画图像来响应手势。
  • Handing Touches Using Responder Methods and Gesture Recognizers includes two projects that demonstrate how to handle multiple touches to drag squares(平方，方形，方块) around onscreen.One version uses gesture recognizers,and the other uses custom touch-event handling methods.The latter version is especially userful for understanding touch phasses because it displays the current touch phase onscreen as the touches occur.
    Handing Touches Using Responder Methods and Gesture Recognizers包括两个工程来说明如何处理多点触摸来在屏幕上拖动方块。一个版本使用手势识别器，另一个使用自定义的时间处理方 法。后边的版本对于理解触摸阶段特别有用，因为当触摸发生时它在屏幕上展示了当前触摸阶段
  • MoveMe shows how to animate a view in response to touch events.Examine(仔细检查) this sample project to further your understanding of custom touch-event handling.
    MoveMe 展示了如何对view动画来响应触摸事件。仔细看这个简单的项目来加深你对自定义触摸事 件处理的理解。
